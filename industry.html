Deduplication with ROW_NUMBER
Keep latest row per customer email
Dedup preview
WITH ranked AS (
  SELECT *,
         ROW_NUMBER() OVER (PARTITION BY email ORDER BY customer_id DESC) AS rn
  FROM customers
)
SELECT customer_id, full_name, email, rn
FROM ranked
ORDER BY email, rn;

Explanation

  1. Partition groups duplicate rows by email.
  2. Highest customer_id is treated as latest row.
  3. rn = 1 means keep row; rn > 1 means duplicate.
15
Funnel analysis
Order status share of total
Funnel report
SELECT
  status,
  COUNT(*) AS order_count,
  SUM(total_amount) AS revenue,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_of_orders
FROM orders
GROUP BY status
ORDER BY order_count DESC;

Explanation

  1. First aggregation groups data by each status stage.
  2. SUM(COUNT(*)) OVER () calculates overall total across groups.
  3. Percentage reveals where most orders are concentrated.
16
Cohort retention
How long customers stay active after signup
Cohort matrix
WITH cohorts AS (
  SELECT customer_id, STRFTIME('%Y-%m', created_at) AS cohort_month
  FROM customers
), activity AS (
  SELECT c.cohort_month, o.customer_id,
         CAST((STRFTIME('%Y', o.order_date) - STRFTIME('%Y', c.cohort_month)) * 12
             + (STRFTIME('%m', o.order_date) - STRFTIME('%m', c.cohort_month)) AS INTEGER) AS months_since_signup
  FROM orders o
  JOIN cohorts c ON o.customer_id = c.customer_id
  WHERE o.status = 'delivered'
)
SELECT cohort_month, months_since_signup, COUNT(DISTINCT customer_id) AS active_customers
FROM activity
GROUP BY cohort_month, months_since_signup
ORDER BY cohort_month, months_since_signup;

Explanation

  1. Create signup cohort (month of registration).
  2. Measure months between signup and each delivered order.
  3. Group to see how many users remain active over time.
17
RFM segmentation
Rank customers by recency, frequency and money
RFM scoring
WITH rfm AS (
  SELECT customer_id,
         JULIANDAY('2023-12-01') - JULIANDAY(MAX(order_date)) AS recency,
         COUNT(*) AS frequency,
         SUM(total_amount) AS monetary
  FROM orders
  WHERE status != 'cancelled'
  GROUP BY customer_id
), scored AS (
  SELECT *,
         4 - NTILE(3) OVER (ORDER BY recency DESC) AS r,
         NTILE(3) OVER (ORDER BY frequency) AS f,
         NTILE(3) OVER (ORDER BY monetary) AS m
  FROM rfm
)
SELECT customer_id, r, f, m, (r+f+m) AS total_score
FROM scored
ORDER BY total_score DESC;

Explanation

  1. Build base metrics per customer.
  2. Use NTILE(3) to split each metric into low/mid/high score.
  3. Combined score helps prioritize campaign targeting.
18
LTV and churn flags
Detect inactive customers and value gap
LTV + churn
WITH metrics AS (
  SELECT customer_id,
         SUM(total_amount) AS lifetime_value,
         MAX(order_date) AS last_order,
         CAST(JULIANDAY('2023-12-01') - JULIANDAY(MAX(order_date)) AS INTEGER) AS days_inactive
  FROM orders
  WHERE status != 'cancelled'
  GROUP BY customer_id
)
SELECT customer_id, lifetime_value, days_inactive,
       CASE
         WHEN days_inactive <= 30 THEN 'Active'
         WHEN days_inactive <= 60 THEN 'At Risk'
         ELSE 'Churned'
       END AS churn_status
FROM metrics
ORDER BY lifetime_value DESC;

Explanation

  1. Calculate each customer’s total value and last activity date.
  2. Compute inactivity days relative to reference date.
  3. CASE labels customer into operational churn buckets.

Funnel Analysis: Order Status Breakdown
See conversion at each stage
Challenge: Analyze the order pipeline:• Count of orders per status• Percentage of total orders for each status• Sort by count (highest first)
Hint: GROUP BY status, COUNT orders, use SUM(COUNT(*)) OVER () to get total for percentage calculation.
Show Answer
SELECT 
  status,
  COUNT(order_id) AS order_count,
  ROUND(COUNT(order_id) * 100.0 / SUM(COUNT(order_id)) OVER (), 1) AS percentage
FROM orders
GROUP BY status
ORDER BY order_count DESC;
Q2
Customer Churn Health Score
Segment customers by inactive days
Challenge: For each customer, calculate:• Total spending (lifetime value)• Days since last order (using reference date 2023-12-01)• Churn status: "Active" (<=30 days), "At Risk" (31-60 days), or "Churned" (>60 days) Show top 10 customers by lifetime value.
Hint: Use CTE to calculate max(order_date), compare to reference. CASE for status. Limit output.
Show Answer
WITH customer_metrics AS (
  SELECT 
    c.full_name,
    SUM(o.total_amount) AS lifetime_value,
    MAX(o.order_date) AS last_order_date,
    CAST(JULIANDAY('2023-12-01') - JULIANDAY(MAX(o.order_date)) AS INTEGER) AS days_inactive
  FROM customers c
  LEFT JOIN orders o ON c.customer_id = o.customer_id
  WHERE o.status IS NULL OR o.status != 'cancelled'
  GROUP BY c.customer_id, c.full_name
)
SELECT 
  full_name,
  ROUND(lifetime_value, 2) AS lifetime_value,
  days_inactive,
  CASE
    WHEN days_inactive <= 30 THEN 'Active'
    WHEN days_inactive <= 60 THEN 'At Risk'
    ELSE 'Churned'
  END AS churn_status
FROM customer_metrics
ORDER BY lifetime_value DESC
LIMIT 10;
