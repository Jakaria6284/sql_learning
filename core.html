6
INNER JOIN and LEFT JOIN
Merge related tables correctly
Employee + department report
SELECT e.full_name, e.salary, d.dept_name, d.location
FROM employees e
INNER JOIN departments d ON e.department_id = d.department_id
ORDER BY d.dept_name, e.salary DESC;

Explanation

  1. INNER JOIN returns only employees that have a matching department.
  2. ON defines relationship key between tables.
  3. Sorting groups people by department and salary rank.
All products including unsold
SELECT p.product_name, COALESCE(SUM(oi.quantity),0) AS units_sold
FROM products p
LEFT JOIN order_items oi ON p.product_id = oi.product_id
GROUP BY p.product_id, p.product_name
ORDER BY units_sold DESC;

Explanation

  1. LEFT JOIN keeps all products even with no matching order items.
  2. SUM becomes NULL when there is no match, COALESCE turns it into 0.
  3. Essential for stock review and unsold item detection.
7
GROUP BY + aggregates
Summarize many rows into business metrics
Department payroll summary
SELECT d.dept_name,
       COUNT(e.employee_id) AS headcount,
       SUM(e.salary) AS payroll,
       ROUND(AVG(e.salary),2) AS avg_salary
FROM departments d
LEFT JOIN employees e ON d.department_id = e.department_id
GROUP BY d.department_id, d.dept_name
ORDER BY payroll DESC;

Explanation

  1. GROUP BY creates one row per department.
  2. COUNT, SUM, AVG calculate key metrics for each group.
  3. Include department id in GROUP BY for stable grouping key.
8
HAVING
Filter grouped results after aggregation
Only busy customers
SELECT c.full_name, COUNT(o.order_id) AS order_count
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.full_name
HAVING COUNT(o.order_id) > 1
ORDER BY order_count DESC;

Explanation

  1. WHERE filters rows before grouping, HAVING filters groups after.
  2. This keeps only customers whose aggregated count is more than one.
9
CASE expression
Add IF/ELSE style logic in SQL
Salary banding
SELECT full_name, salary,
  CASE
    WHEN salary >= 100000 THEN 'Senior / Executive'
    WHEN salary >= 75000 THEN 'Mid-Level'
    ELSE 'Entry'
  END AS salary_band
FROM employees
ORDER BY salary DESC;

Explanation

  1. CASE evaluates rules top to bottom.
  2. First matching condition returns corresponding label.
  3. Useful for bucketing and decision logic in reports.
10
Subqueries
Use one query as input for another
Above-average salary
SELECT full_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees)
ORDER BY salary DESC;

Explanation

  1. Inner query computes company average once.
  2. Outer query compares each employee salary with that value.
  3. This pattern is common for benchmark filtering.

Q1
Multi-Table Revenue Report
JOINs with aggregation
Challenge: Show each product category with total revenue (sum of quantity × price) from all orders. Sort by revenue (highest first).
Hint: JOIN products → order_items, GROUP BY category, SUM(quantity * price), ORDER BY revenue DESC.
Show Answer
SELECT 
  p.category,
  SUM(oi.quantity * p.unit_price) AS total_revenue
FROM products p
JOIN order_items oi ON p.product_id = oi.product_id
GROUP BY p.category
ORDER BY total_revenue DESC;
Q2
Filter Groups with HAVING
Departments with high average salary
Challenge: Find departments where the average employee salary is above $70,000. Show department name and average salary.
Hint: GROUP BY department, use HAVING AVG(salary) > 70000, no WHERE needed here.
Show Answer
SELECT 
  d.dept_name,
  ROUND(AVG(e.salary), 2) AS avg_salary
FROM departments d
LEFT JOIN employees e ON d.department_id = e.department_id
GROUP BY d.department_id, d.dept_name
HAVING AVG(e.salary) > 70000
ORDER BY avg_salary DESC;
