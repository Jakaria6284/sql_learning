CTEs (WITH)
Break big query into clear steps
VIP customers with CTE
WITH customer_spend AS (
  SELECT customer_id, SUM(total_amount) AS total_spend
  FROM orders
  GROUP BY customer_id
)
SELECT c.full_name, cs.total_spend
FROM customer_spend cs
JOIN customers c ON cs.customer_id = c.customer_id
WHERE cs.total_spend > 300
ORDER BY cs.total_spend DESC;

Explanation

  1. CTE computes spend once and gives it a reusable name.
  2. Main query stays short and readable.
  3. Best practice when query has multiple logical steps.
12
Window functions
Calculate rankings and trends without collapsing rows
Top 2 earners per department
WITH ranked AS (
  SELECT e.full_name, d.dept_name, e.salary,
         DENSE_RANK() OVER (
           PARTITION BY e.department_id
           ORDER BY e.salary DESC
         ) AS rnk
  FROM employees e
  JOIN departments d ON e.department_id = d.department_id
)
SELECT dept_name, full_name, salary, rnk
FROM ranked
WHERE rnk <= 2
ORDER BY dept_name, rnk;

Explanation

  1. PARTITION BY resets ranking inside each department.
  2. DENSE_RANK keeps ties at same rank without gaps.
  3. Filter rnk <= 2 to keep top-N per group.
Month over month change with LAG
WITH monthly AS (
  SELECT STRFTIME('%Y-%m', order_date) AS month,
         SUM(total_amount) AS revenue
  FROM orders
  WHERE status = 'delivered'
  GROUP BY STRFTIME('%Y-%m', order_date)
)
SELECT month, revenue,
       LAG(revenue) OVER (ORDER BY month) AS prev_month,
       ROUND((revenue - LAG(revenue) OVER (ORDER BY month)) * 100.0 /
             NULLIF(LAG(revenue) OVER (ORDER BY month), 0), 1) AS mom_pct
FROM monthly
ORDER BY month;

Explanation

  1. LAG fetches previous row’s revenue for comparison.
  2. Formula computes month-over-month percentage change.
  3. NULLIF(...,0) prevents divide-by-zero errors.
13
INSERT / UPDATE / DELETE safely
Write operations with preview-first habit
Insert and verify
INSERT INTO products (product_id, product_name, category, unit_price, stock_qty)
VALUES (16, 'Standing Desk Pro', 'Accessories', 399.99, 30);

SELECT * FROM products WHERE product_id = 16;

Explanation

  1. INSERT adds new row into target table.
  2. Immediate SELECT confirms write happened correctly.
Update with preview
SELECT product_name, unit_price
FROM products
WHERE category = 'Electronics' AND stock_qty > 200;

-- UPDATE products
-- SET unit_price = unit_price * 0.90
-- WHERE category = 'Electronics' AND stock_qty > 200;

Explanation

  1. Run SELECT first to preview affected rows.
  2. If preview is correct, execute UPDATE section.
  3. This prevents accidental mass changes in production.
💡 Safe rule: before every UPDATE/DELETE, run the same WHERE in a SELECT COUNT(*) preview.
