19
Easy Challenge
Electronics list sorted by price
Solution
SELECT product_name, unit_price, stock_qty
FROM products
WHERE category = 'Electronics'
ORDER BY unit_price DESC;

Explanation

  1. Filter category first using WHERE.
  2. Sort descending to show most expensive first.
  3. Returns exactly requested output columns.
20
Medium Challenge
Orders per customer (more than one)
Solution
SELECT c.full_name, COUNT(o.order_id) AS order_count
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.full_name
HAVING COUNT(o.order_id) > 1
ORDER BY order_count DESC;

Explanation

  1. JOIN links customer names to order rows.
  2. GROUP BY creates one row per customer.
  3. HAVING keeps only grouped customers with order count > 1.

Hard Challenge
Department budget status using CASE
Solution
WITH payroll AS (
  SELECT department_id, COUNT(*) AS headcount, SUM(salary) AS total_payroll
  FROM employees
  GROUP BY department_id
)
SELECT d.dept_name, p.headcount, p.total_payroll, d.budget,
       d.budget - p.total_payroll AS remaining,
       CASE
         WHEN p.total_payroll > d.budget * 1.05 THEN 'OVER BUDGET'
         WHEN p.total_payroll < d.budget * 0.95 THEN 'UNDER BUDGET'
         ELSE 'ON TARGET'
       END AS budget_status
FROM departments d
JOIN payroll p ON d.department_id = p.department_id
ORDER BY remaining;

Explanation

  1. CTE calculates payroll once per department.
  2. Main query compares payroll with budget.
  3. CASE applies ±5% rule for status labels.
