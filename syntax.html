SELECT + FROM
Tutorial: read columns from a table

Tutorial: SELECT chooses columns. FROM chooses source table.

Question: Show product name, category, and price for all products.

SQL Answer
SELECT product_name, category, unit_price
FROM products;

Explanation

  1. Only selected columns are returned, not full row.
  2. This keeps results clean and faster than SELECT * in production.
2
WHERE
Tutorial: filter rows by condition

Question: Find electronics products that cost more than 80.

SQL Answer
SELECT product_name, unit_price
FROM products
WHERE category = 'Electronics' AND unit_price > 80;

Explanation

  1. WHERE runs before sorting/grouping and keeps only matching rows.
  2. AND means both conditions must be true.
3
DISTINCT
Tutorial: unique values only

Question: List unique customer countries.

SQL Answer
SELECT DISTINCT country
FROM customers;

Explanation

  1. DISTINCT removes duplicates after reading rows.
  2. Useful for filters, dropdown options, and profile summaries.
4
ORDER BY + LIMIT
Tutorial: sort and top-N

Question: Show top 5 most expensive products.

SQL Answer
SELECT product_name, unit_price
FROM products
ORDER BY unit_price DESC
LIMIT 5;

Explanation

  1. DESC means high to low.
  2. LIMIT 5 keeps output concise for ranking use-cases.
5
LIKE / IN / BETWEEN
Tutorial: pattern/list/range filtering

Question: Find products with “Keyboard” in name and customers from USA/Canada.

SQL Answer A (LIKE)
SELECT product_name, unit_price
FROM products
WHERE product_name LIKE '%Keyboard%';

Explanation

  1. % matches any characters before/after the word.
SQL Answer B (IN)
SELECT full_name, country
FROM customers
WHERE country IN ('USA','Canada');

Explanation

  1. IN is cleaner than many OR conditions.
6
INNER JOIN
Tutorial: combine matching rows from two tables

Question: Show each employee with department name.

SQL Answer
SELECT e.full_name, e.salary, d.dept_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.department_id;

Explanation

  1. JOIN key is department id in both tables.
  2. Rows without match are excluded in INNER JOIN.
7
LEFT JOIN
Tutorial: keep all left-table rows

Question: Show all products and sold quantity (include unsold).

SQL Answer
SELECT p.product_name, COALESCE(SUM(oi.quantity),0) AS units_sold
FROM products p
LEFT JOIN order_items oi ON p.product_id = oi.product_id
GROUP BY p.product_id, p.product_name;

Explanation

  1. LEFT JOIN preserves every product row.
  2. COALESCE turns NULL sold count into 0.
8
GROUP BY + Aggregate
Tutorial: summarize by group

Question: Department headcount and payroll.

SQL Answer
SELECT d.dept_name,
       COUNT(e.employee_id) AS headcount,
       SUM(e.salary) AS total_payroll
FROM departments d
LEFT JOIN employees e ON d.department_id = e.department_id
GROUP BY d.department_id, d.dept_name;

Explanation

  1. Each department becomes one output row.
  2. COUNT and SUM calculate metrics over grouped rows.
9
HAVING
Tutorial: filter groups after aggregate

Question: Show only customers with more than one order.

SQL Answer
SELECT c.full_name, COUNT(o.order_id) AS order_count
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.customer_id, c.full_name
HAVING COUNT(o.order_id) > 1;

Explanation

  1. HAVING is like WHERE for grouped rows.
  2. Use HAVING when condition uses aggregate output.
10
CASE WHEN
Tutorial: conditional logic in SQL

Question: Categorize salaries into bands.

SQL Answer
SELECT full_name, salary,
CASE
  WHEN salary >= 100000 THEN 'Senior'
  WHEN salary >= 75000 THEN 'Mid'
  ELSE 'Entry'
END AS salary_band
FROM employees;

Explanation

  1. Rules run from top to bottom.
  2. First matched condition decides label.
11
Subquery
Tutorial: use query inside query

Question: Employees above company average salary.

SQL Answer
SELECT full_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

Explanation

  1. Inner query returns one value: average salary.
  2. Outer query compares each row with that value.
12
CTE (WITH)
Tutorial: readable multi-step query

Question: Find VIP customers with total spend above 300.

SQL Answer
WITH spend AS (
  SELECT customer_id, SUM(total_amount) AS total_spend
  FROM orders
  GROUP BY customer_id
)
SELECT customer_id, total_spend
FROM spend
WHERE total_spend > 300;

Explanation

  1. CTE creates temporary named result.
  2. Main query reads from it like a table.
13
Window Function (DENSE_RANK)
Tutorial: rank rows without collapsing

Question: Top 2 earners in each department.

SQL Answer
WITH ranked AS (
  SELECT e.full_name, e.department_id, e.salary,
         DENSE_RANK() OVER (PARTITION BY e.department_id ORDER BY e.salary DESC) AS rnk
  FROM employees e
)
SELECT *
FROM ranked
WHERE rnk <= 2;

Explanation

  1. PARTITION BY resets rank per department.
  2. Window function keeps every row and adds rank column.
14
INSERT / UPDATE / DELETE
Tutorial: writing data safely

Question: Add one product, then preview update and delete targets safely.

SQL Answer
INSERT INTO products (product_id, product_name, category, unit_price, stock_qty)
VALUES (16, 'Standing Desk Pro', 'Accessories', 399.99, 30);

SELECT * FROM products WHERE product_id = 16;

-- Preview rows before update/delete
SELECT COUNT(*) AS to_update FROM products WHERE category = 'Electronics';
SELECT COUNT(*) AS to_delete FROM orders WHERE status = 'cancelled';

Explanation

  1. Always verify with SELECT/COUNT before UPDATE/DELETE.
  2. Safe preview prevents accidental mass data changes.
